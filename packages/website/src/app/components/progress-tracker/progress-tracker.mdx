## Tracker

<div className="admiralty-intro">
  Use the tracker component to help users understand progress through a service or the current status of a task.
  Trackers should reduce uncertainty and help users plan what to do next.
</div>

<hr style={{ border: "1px solid #e0e0e0" }} />

## When to use this component

<div style={{ paddingBottom: "30px" }}>

Use a tracker when users need to:

- Understand where they are in a multi-step journey
- See which sections are complete or incomplete
- Check the status of an application or request
- Return later and continue a task

</div>

<div style={{ paddingBottom: "30px" }}>
Do not use a tracker:

- For short, simple journeys (1 to 3 screens)
- If there is no meaningful concept of progress
- As a visual embellishment

</div>

## Types of tracker

<div>Choose the pattern that matches the user need.</div>

### Step tracker (linear journeys)

<div style={{ paddingBottom: "30px" }}>
  Use for multi-step forms where users complete sections in order.

The step tracker:

- Shows the total number of steps
- Clearly identifies the current step
- Uses plain English labels
- Does not allow users to skip ahead unless the service supports it
- Allows user to step back in the journey to return to a previous step to amend cached data

</div>

<div style={{ paddingBottom: "30px" }}>
  Do not show page validation errors in the tracker. Use an error summary and inline validation instead.
</div>

### Task tracker (checklist style)

<div style={{ paddingBottom: "30px" }}>
  Use when users can complete sections in any order or return later.

The task tracker shows the status of each section. For example:

- Account creation
- Payment setup
- Verification

</div>

<div style={{ paddingBottom: "30px" }}>
  Use clear text labels. Do not rely on colour alone to communicate status.
  
  Avoid using the word "Error" unless something has genuinely failed.

</div>

### Status tracker (system-driven progress)

<div style={{ paddingBottom: "30px" }}>
  Use when the tracker reflects the system's progress, not the user's progress.

For example:

- Submitted
- Under review
- More information needed
- Approved

</div>

<div>
  If user action is required, clearly explain what they must do next.

</div>

## Content design

<div style={{ paddingBottom: "30px" }}>
  Follow plain English principles.

- Use short, descriptive step names
- Avoid internal or technical terminology
- Keep labels consistent throughout the journey
- Do not change step names part way through a service

</div>

<div style={{ paddingBottom: "30px" }}>Avoid motivational or celebratory language. Keep tone factual and calm.</div>

## Accessibility

<div style={{ paddingBottom: "30px" }}>
  The component provides step information through ARIA labels and semantic HTML. Each step is marked with its status
  (complete, current, or upcoming) to help assistive technology users understand the progress flow.
</div>
